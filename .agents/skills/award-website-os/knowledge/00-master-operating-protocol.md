# 00 — Master Operating Protocol

This is the governing protocol for building moon-level websites from minimal input.

## Reality Check

No static prompt file can contain "all possible data" in the literal sense. A real master system must combine:

1. Strong resident knowledge in agents.
2. Deep local reference files.
3. Current web research.
4. Project files and user assets.
5. Browser/screenshot inspection.
6. QA tools.
7. Real specialist workflows.
8. Human approval for money, credentials, publishing, and legal risk.

This OS is designed as a practical master bench: it gives Codex/Claude the roles, operating rules, reference knowledge, tool-routing logic, and ship gates needed to act like a senior creative-technical studio.

## The One-Input Rule

The user may provide only a business type, for example:

> "luxury dental clinic"
> "construction SaaS"
> "architecture studio"
> "AI recruiting platform"

The agent must proceed by inference and research, not by asking 15 questions.

Ask only if:
- payment is required
- credentials are required
- legal/medical/financial claims need verification
- brand assets must be exact
- publishing/deployment would affect a real account
- the user explicitly requests a specific visual identity but omits mandatory assets

## Required Sequence

1. Interpret the business type.
2. Research market and competitor patterns.
3. Define business goal and conversion.
4. Generate 3 concept territories.
5. Select strongest concept.
6. Define art direction.
7. Write copy and page structure.
8. Plan assets.
9. Generate/source/create assets legally.
10. Choose technical stack.
11. Build first shippable version.
12. Add meaningful motion.
13. Add 3D/WebGL only if justified.
14. Inspect screenshots.
15. Run QA.
16. Fix issues.
17. Final handoff.

## Hard Ship Gates

Do not ship if:
- business value is unclear in 5 seconds
- site looks like a template
- typography is default/generic
- layout is repetitive and weak
- mobile is squeezed desktop
- nav is confusing
- CTA is missing or vague
- animations delay comprehension
- 3D is slow or meaningless
- images are unlicensed or misleading
- generated people are presented as real
- fake metrics/testimonials/logos are used
- keyboard navigation fails
- contrast is weak
- build/typecheck/lint fail
- console has serious errors

## Autonomous Tool Policy

Use tools aggressively but legally:

Allowed:
- web research
- browser testing
- screenshot inspection
- local scripts
- image generation with authorized API/tool
- Blender if installed
- Playwright/Lighthouse/axe if installed
- MCP tools if configured
- stock asset APIs if credentials are provided
- deployment only with approval

Forbidden:
- bypassing paywalls
- pirating assets
- using copyrighted images without permission
- purchasing subscriptions without approval
- exposing secrets
- publishing without approval
- impersonating real users/customers
- fake social proof
