# 27 — The No-Template Checklist

Run this before final delivery.

## Hero

- [ ] Headline is specific to the business.
- [ ] Hero composition is not generic centered text + button.
- [ ] There is a memorable visual object or typographic move.
- [ ] CTA is clear.
- [ ] Above fold passes 5-second test.
- [ ] Mobile hero is intentionally composed.

## Copy

- [ ] No vague "innovative/seamless/next-gen" language unless justified.
- [ ] Every section answers a user doubt.
- [ ] Claims are proof-backed or softened.
- [ ] No fake metrics.
- [ ] No fake testimonials.
- [ ] CTA labels are action-specific.

## Visual System

- [ ] Typography is customized.
- [ ] Color tokens are intentional.
- [ ] Layout rhythm varies.
- [ ] Icons/illustrations share a style.
- [ ] Motif repeats across sections.
- [ ] Footer is designed.

## Interaction

- [ ] One signature interaction exists if the project aims high.
- [ ] Motion has a role.
- [ ] Reduced motion works.
- [ ] No hover-only essential content.
- [ ] No scroll-jacking.

## Assets

- [ ] Asset ledger exists.
- [ ] Licenses are known.
- [ ] Generated assets are not misleading.
- [ ] Hero asset optimized.
- [ ] Mobile crops/fallbacks exist.
- [ ] 3D is justified if used.

## Engineering

- [ ] Build passes.
- [ ] Typecheck passes.
- [ ] Lint passes.
- [ ] No console errors.
- [ ] Metadata exists.
- [ ] 404 exists.
- [ ] Form states exist if forms exist.

## Accessibility

- [ ] Keyboard nav works.
- [ ] Focus visible.
- [ ] Contrast checked.
- [ ] Forms labeled.
- [ ] Alt text correct.
- [ ] Target sizes OK.
- [ ] Reduced motion respected.

## Final Question

If the logo were removed, would the site still feel like it belongs to this specific business?

If no, it is still too generic.
