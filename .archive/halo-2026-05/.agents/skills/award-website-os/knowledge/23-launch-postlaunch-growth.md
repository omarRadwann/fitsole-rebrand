# 23 — Launch and Post-Launch Growth

## Mission

The website should not only look good; it should produce results.

## Launch Checklist

- domain connected
- SSL active
- analytics installed
- error tracking installed
- sitemap generated
- robots configured
- metadata complete
- OG image tested
- favicon/app icons
- forms tested
- email/CRM integration tested
- WhatsApp/phone links tested
- page speed checked
- 404 designed
- privacy/legal pages if needed
- cookie/consent if needed

## Analytics Events

Track:
- primary CTA click
- secondary CTA click
- form start
- form submit success
- form submit error
- demo/booking click
- WhatsApp click
- phone click
- pricing viewed
- case study opened
- video played if important
- scroll depth only if useful

## Conversion Improvements

After launch, test:
- hero headline
- CTA text
- proof placement
- form length
- WhatsApp vs form
- pricing/process clarity
- testimonial/case proof
- mobile CTA sticky behavior

## SEO Basics

For local service:
- LocalBusiness schema
- service pages
- city/location page if real
- phone/address/hours
- FAQs
- reviews if real
- map if appropriate

For SaaS:
- use-case pages
- comparison pages
- integrations
- feature pages
- docs/blog if strategy supports it

For portfolio:
- case studies
- project metadata
- process pages
- image alt
- service keywords

## Post-Launch QA

After deployment:
- test production URL
- test mobile real device
- test form delivery
- test analytics events
- check console errors
- run Lighthouse production
- inspect social share preview
- verify no staging secrets/content
