# 14 — AI Anti-Genericity Protocol

AI tends to produce average output unless forced away from defaults.

## The Genericity Trap

AI defaults to:
- centered hero
- gradient background
- rounded cards
- vague headline
- generic icons
- fake dashboard
- "trusted by teams"
- "streamline your workflow"
- overused reveal animation
- no real art direction

## Anti-Genericity Workflow

### Step 1 — Ban the Defaults

Before concepting, list banned patterns:
- centered hero unless justified
- 3-card feature grid unless content demands it
- generic testimonial blocks
- gradient blob
- fake product dashboard
- random Spline object
- stock team photo
- default shadcn visual style
- "Learn more" primary CTA

### Step 2 — Force Specificity

For every section, define:
- what user doubt it answers
- what proof or detail supports it
- what visual metaphor it uses
- what action it moves toward

### Step 3 — Use the "Only This Business" Test

Ask:
```txt
Could this section exist unchanged on another site?
```

If yes, rewrite/redesign.

### Step 4 — Replace Abstract Claims

Bad:
"Powerful tools for modern teams."

Better:
"Know which job is blocked, who owns it, and what material is missing before the morning call."

### Step 5 — Make Visuals Functional

Every visual should do at least one:
- explain process
- create trust
- show product
- express metaphor
- create emotional desire
- provide proof
- orient the user

## Anti-AI Language Rewrite Table

| Generic | Better |
|---|---|
| streamline workflows | remove the daily handoff chaos |
| powerful platform | one place to see what changed, who owns it, and what is blocked |
| next-gen experience | a quieter way to book, understand, and trust the process |
| innovative solutions | a process built around [specific user pain] |
| scalable and robust | handles [specific scale/context] without [specific failure] |
| seamless | describe the actual transition or workflow |
| unlock potential | name the outcome |
| tailored | explain how it adapts |

## Section Smell Test

Flag a section if:
- headline could fit any company
- icon is generic
- card title has one word
- paragraph uses abstract nouns
- visual is decorative only
- no user objection is answered
- no new information appears
- CTA is absent
- layout repeats previous section too closely

## The "Cut 30%" Rule

After the first build:
1. Cut 30% of weak copy.
2. Cut 30% of unnecessary cards.
3. Cut 30% of decorative motion.
4. Increase strength of remaining 70%.

## Codex Prompt to Use

```txt
Audit this site for AI-generic patterns.
Find every section that could be pasted into another website.
For each, rewrite the copy, redesign the visual idea, or delete it.
Do not add more effects. Increase specificity.
```
