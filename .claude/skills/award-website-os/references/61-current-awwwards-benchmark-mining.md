# 61 — Current Awwwards Benchmark Mining

How to mine current Awwwards / SOTD / FWA / CSSDA / SiteInspire benchmarks without copying them. Use this when starting a project or before the concept competition.

## What to mine, in order

1. **Category winners (last 18 months)** — sites in the same industry that won an Awwwards SOTD or honorable mention. Capture the signature interaction, the hero composition, and the type system.
2. **Out-of-category winners** — the strongest 5 sites in other industries from the same period. Capture motion language, structural ideas, and craft details.
3. **Studio portfolios** — 3–5 senior studios that work in this category (e.g. Locomotive, Resn, Active Theory, Studio Liberal, Hello Monday, North Kingdom, Tundra, Immersive Garden, Lusion, FFFFFF.studio). Note what they reuse and what they would never do.
4. **Anti-references** — 3–5 sites in the same category that look generic. Note why. This becomes the cliché list in `docs/research-brief.md`.
5. **Type/motion/3D specialists** — sites praised specifically for typography, scroll, or WebGL. Mine the technique, not the look.

## Capture format (`docs/benchmark-reference-board.md`)

For each reference, record:

```
URL:
Category:
Why it won (in one sentence):
Signature idea:
Signature interaction:
Hero composition (in 3 lines):
Type system:
Motion language:
3D / WebGL role:
What we'd steal (technique, never look):
What we won't repeat:
```

10 references in the board is plenty. More than 20 is a tell that the concept hasn't been chosen yet.

## The borrowing rules

- Steal **techniques**, not surfaces. A scroll mechanic, a reveal pattern, a shader uniform setup — yes. A specific palette, an exact type pairing, a copied animation — no.
- Never copy proprietary 3D objects, branded characters, or signature interactions that are clearly the studio's identity.
- If a reference's craft cannot be reproduced under this project's time/skill budget, drop it and find one whose craft is within reach.

## How to use the board during concept competition

For each of the three concepts (safe / experimental / moonshot), name which references inform it and which references it explicitly rejects. A concept that cites no rejections is usually a concept that is too close to its references.

## Anti-mining traps

- "Inspiration boards" with 60 screenshots and no decision.
- Pinning a famous site as the look without understanding why it works.
- Mining only category winners (the result will be a category cliché).
- Mining only motion/3D demos (the result will be a developer demo, not a site).

## The mining test

After the board is built, answer:

```
If we delivered this project, which of these references would the user be
most upset to see Awwwards link to ours?
```

If the answer is "any of them," the concept is too close. Push further away.
