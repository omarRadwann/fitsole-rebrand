# Signature Interaction and Game Feel Master

## Definition

A signature interaction is a memorable behavior that helps the user understand, trust, desire, or act. It is not decorative animation.

## The five-part spec

Every signature interaction must define:

1. business meaning
2. user input
3. system response
4. emotional effect
5. conversion role

## Game-feel details

Borrow from games carefully:

- immediate response to input
- anticipation before large movement
- inertia/deceleration instead of dead linear movement
- satisfying completion state
- clear affordance before interaction
- layered feedback: visual first, optional sound/haptic-like cues only when appropriate
- forgiving interaction: works with mouse, touch, keyboard, and reduced motion

## Good examples by business type

- logistics: tangled routes resolve into a calm network
- construction: project layers assemble from ground to finish
- education: skill blocks combine into a learner path
- healthcare: complex patient journey simplifies into clear next steps
- agency: messy brief transforms into brand system
- e-commerce: product material/fit/story becomes explorable

## Bad examples

- orbiting sphere with no message
- cursor blob over every element
- scroll-jack just to show off
- generic particles behind SaaS copy
- 3D object unrelated to the service

## Acceptance criteria

The interaction passes only if a non-designer can answer: “what did that interaction tell me about the business?”
