# Super Strong Design Playbook for Claude Code

Use this as the practical design brain of the pack. The goal is to force Claude Code away from safe template output and toward specific, polished, commercially useful work.

## What strong design means here

Strong design is the intersection of:

- Strategic clarity: the site knows what it is selling and to whom.
- Visual memorability: the site has a signature, not just nice spacing.
- Craft: typography, spacing, motion, assets, and responsiveness feel intentional.
- Proof: beautiful claims are backed by evidence.
- Speed and accessibility: the site feels premium without punishing users.

## The 30-minute design sequence Claude Code must follow

1. Read the business type, current site, files, or user brief.
2. Write a one-sentence positioning hypothesis.
3. Collect or infer 6 reference types: category, out-of-category, typography, motion, 3D, conversion.
4. Identify clichés to avoid.
5. Create three concepts: safe premium, distinctive, moonshot.
6. Score each concept for design, UX, creativity, content, craft, business fit, and risk.
7. Select one concept and explain why.
8. Define type, color, layout, imagery, motion, and 3D rules.
9. Build only after the direction is clear.
10. Screenshot, critique, and improve.

## Concept quality gates

A concept is weak if it can fit any business after swapping the logo. A concept is strong if it can be explained as:

```txt
This business is about [truth], so the website behaves like [visual metaphor] and proves it through [specific proof/interaction].
```

## The signature interaction ladder

Choose the lowest-weight interaction that creates memorability:

1. Typographic reveal.
2. Editorial image crop and scroll rhythm.
3. Cursor or hover behavior.
4. Process animation.
5. Data or product visualization.
6. Lightweight shader texture.
7. R3F object or scene.
8. Spline scene integration.
9. Blender-generated GLB hero asset.
10. Full WebGL experience.

Do not jump to level 10 when level 3 would be stronger and faster.

## Design patterns worth using

- Editorial split hero: large type, controlled image, strong proof cue.
- Cinematic scroll chapter: one story broken into scenes.
- Product-as-monument: hero object treated like a luxury artifact.
- Process machine: animated flow that proves the operating system.
- Data landscape: real metrics turned into spatial or visual form.
- Material study: macro textures, lighting, surfaces, and close-ups.
- Founder-led proof: human credibility presented with editorial restraint.
- Before-after reveal: transformation shown with interaction.

## Anti-template triggers

Claude Code must stop and redesign if it sees:

- Same card grid repeated in more than two sections.
- Generic icons instead of brand-specific visuals.
- Hero headline that could describe any company.
- Random 3D object unrelated to the offer.
- Purple/blue gradient used as the main personality.
- All sections centered with equal spacing.
- Animation applied to every element equally.
- Stock photo that does not add proof.
- CTA text that does not clarify next action.

## Layout craft rules

- Set a max text width for readable copy.
- Use one very large type moment and several quiet sections.
- Create section-to-section contrast: scale, density, media, or rhythm.
- Respect mobile first; design the mobile hero before polishing desktop.
- Every grid needs a reason: comparison, proof, process, options, or stories.
- Use borders, shadows, blur, and glass sparingly.

## 3D/WebGL craft rules

- Keep the camera path simple and purposeful.
- Use lighting to reveal material and hierarchy.
- Compress and optimize every GLB.
- Cap post-processing on mobile.
- Pause expensive animation when offscreen.
- Provide reduced-motion and static poster states.
- Dispose geometries/materials and avoid memory leaks.
- Use instancing for repeated meshes.
- Prefer baked textures/lightmaps when visual richness matters.
- Do not use 3D to compensate for weak strategy.

## Screenshot critique script

After screenshots, Claude Code must answer:

1. What is the first thing the eye sees?
2. Is the promise readable in under five seconds?
3. What feels specific to this business?
4. What feels like a template?
5. Does mobile keep the same idea or only stack boxes?
6. Does motion help or distract?
7. What one change would increase taste most?

## Final design standard

The final page should have at least one frame that could be posted as the brand's visual identity. If no screenshot feels memorable, the design is not finished.
